# Key Points

* **EOL Systems**
  * Microsoft no longer provides security updates once a system reaches end-of-life.
  * Many organizations still use them due to legacy software or cost limitations.
  * Examples include Windows XP, Windows 7, Windows Server 2003, Windows Server 2008.

* **Impact of EOL**
  * No security patches.
  * Reduced software and hardware support.
  * High exposure to known, unpatched vulnerabilities.

* **Penetration Testing Considerations**
  * Legacy systems tend to be easier to exploit due to missing security features.
  * Always confirm with the client before testing to avoid breaking critical legacy services.
  * Recommend isolating or segmenting EOL hosts.

* **Key Differences**
  * Older versions lack modern protections found in newer Windows systems.
  * This often makes privilege escalation straightforward.
  * Examples of critical historical vulnerabilities: MS08-067, EternalBlue.

* **Importance of Awareness**
  * Testers must understand legacy OS risks and common exploit paths.
  * Knowing EOL dates helps quickly identify weak points.

* Legacy Windows systems introduce serious risk.
